## Resolving Setup and Workflow Issues

If a deployment does not behave as expected, inspect the Radius application graph first. It can reveal missing dependencies, incorrect resource links, or environment assumptions that are difficult to see in isolated configuration files.

When infrastructure output differs between environments, compare Radius recipes and provider settings. Radius Terraform and Radius Bicep definitions should be reviewed alongside environment configuration so teams can separate application intent from provisioning logic.

For Kubernetes connection issues, confirm cluster access, namespace permissions, and controller status before changing app definitions. Radius Kubernetes workflows depend on both the application model and the operational health of the target environment.
